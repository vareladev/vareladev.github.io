En el desarrollo de software moderno, el control de versiones es una herramienta esencial para gestionar cambios en el código, colaborar con otros desarrolladores y mantener un historial claro de las modificaciones realizadas. Git, el sistema de control de versiones más utilizado, junto con GitHub, una plataforma de alojamiento de repositorios, son fundamentales para cualquier programador o equipo de trabajo.

Esta guía práctica te ayudará a:

  • Crear una cuenta en GitHub para alojar tus proyectos.
  • Instalar Git en tu computadora y configurarlo correctamente.
  • Aprender los comandos básicos como git init, git commit y git push, que te permitirán empezar a trabajar con repositorios locales y remotos.

Al finalizar, estarás listo para comenzar el aprendizaje relacionado con la gestión de proyectos de desarrollo de software de manera eficiente, colaborar con otros desarrolladores y aprovechar al máximo las ventajas del control de versiones. Pero antes, es importante conocer algunos conceptos fundamentales.

Concepto

Descripción

Git

Sistema de control de versiones distribuido y que se instala en una computadora. Permite rastrear cambios en archivos y coordinar trabajo entre desarrolladores.

GitHub

Plataforma en línea para alojar repositorios Git, colaborar y compartir código entre colegas y la comunidad.

Repositorio (repo)

Carpeta donde se almacena el código y su historial de versiones. Puede estar en local (Git) o remoto (GitHub).

Commit

Una instantánea de los cambios realizados en el código. Incluye un mensaje que describe los cambios.

Branch (rama)

Línea paralela de desarrollo. Permite trabajar en nuevas características sin afectar el código principal.

Merge

Integración de los cambios de una rama en otra, por lo general en la rama principal (main o master).

Clone

Copia de un repositorio remoto (por ejemplo, desde GitHub) a tu máquina local.

Push

Enviar los cambios realizados localmente al repositorio remoto en GitHub.

Pull

Obtener y fusionar cambios desde el repositorio remoto al repositorio local.

Fork

Copia de un repositorio de otro usuario en tu propia cuenta de GitHub. Útil para proponer cambios sin acceso directo al repo original.

Pull Request (PR)

Solicitud para que los cambios en una rama sean revisados e integrados en otra rama (comúnmente main).

.gitignore

Archivo que especifica qué archivos o carpetas deben ser ignorados por Git (por ejemplo, archivos temporales o binarios).

Status

Comando (git status) que muestra el estado actual de los archivos en el repositorio local.

Log

Comando (git log) que muestra el historial de commits realizados en el repositorio.

Git es un sistema de control de versiones distribuido diseñado para rastrear cambios en archivos de código (o cualquier proyecto) a lo largo del tiempo. Fue creado por Linus Torvalds (el mismo creador de Linux) en 2005 y se ha convertido en el estándar para el desarrollo de software.

¿Para qué se usa Git?

  • Desarrollo de software: Guardar cambios incrementalmente con commits.
  • Trabajo en equipo: Sincronizar código entre múltiples desarrolladores.
  • Experimentar sin riesgo: Crear ramas para probar nuevas ideas sin romper el código estable.
  • Recuperación de errores: Volver a una versión anterior si algo falla.

Instalando Git en Microsoft Windows

1. Descarga Git. Accede al sitio oficial de Git Haz clic en "Download for Windows".

2. Abre el archivo descargado (Git-X.X.XX-64-bit.exe). Sigue los pasos del asistente (en general se usarán las opciones predeterminadas).

3. Paso 1. configuración es seleccionar el directorio donde se instalará git. En general, se deja el directorio por defecto. Se sugiere no cambiarlo.

4. Paso 2. Seleccionar componentes que se instalarán.. Asegurarse que los componentes seleccionados sean los que aparecen en la siguiente figura y hacer clic en continuar.

5. Paso 3. Configurar nombre del directorio en la sección de inicio. Dejar la configuración por defecto.

6. Paso 4. Editor predeterminado utilizado por Git. Elige el editor de texto que se abrirá por defecto cuando Git lo necesite (por ejemplo, al escribir mensajes de confirmación). Recomendado: Visual Studio Code o Vim si estás familiarizado; sino, Notepad++ o Nano son buenas opciones sencillas.

7. Paso 5. Configurar el nombre de la rama inicial en nuevos repositorios. Este paso te permite elegir el nombre de la rama inicial cuando se crea un nuevo repositorio con "git init". Se sugiere seleccionar la opción "Override the default Branch name for new repositories" y asegurarse que el nombre sea "main", esto porque GitHub y muchas otras plataformas usan el nombre "main" por defecto desde 2020. No utilizar esta configuración puede causar serios conflictos en el futuro para utilizar Github.

8. Paso 6. Configurar variables de entorno. Se sugiere seleccionar la opción "Use Git from the command line and also from 3rd-party software" porque permite utilizer git desde otras herramientas como desde CMD o PowerShell.

9. Paso 7. Selección del servicio SSH. En este paso eliges qué programa va a manejar las conexiones SSH cuando uses comandos como git push o git pull por SSH. Se sugiere la opción "Use bundled OpenSSH". Que usa la versión de OpenSSH que viene incluida con la instalación de Git para Windows, que funciona bien en la mayoría de los casos.

10. Paso 8. Seleccionar el backend de transporte HTTPS. Se sugiere seleccionar la opción "Use the OpenSSL library", que usa OpenSSL para manejar conexiones HTTPS. Es estándar y con alto grado de compatibilidad.

11. Paso 9. Configurar la convención de fin de línea. Seleccionar la opción "Checkout Windows-style, commit Unix-style line endings", que convierte los saltos de línea a CRLF al clonar y a LF al confirmar. Ideal para compatibilidad entre sistemas operativos.

12. Paso 10. Configurar el emulador de terminar para usar con Git Bash. Seleccionar "Use MinTTY (the default terminal of MSYS2)". Que es una terminal independiente con mejor soporte para colores, fuentes, y otras herramientas.

13. Paso 11. Seleccionar el comportamiento por defecto del comando "git pull". Selecciona la opción "Default (fast-forward or merge)".

14. Paso 12. Seleccionar el asistente de credenciales. Seleccionar la opción "Git Credential Manager", que maneja credenciales de forma segura, guarda tokens y contraseñas.

15. Paso 13. Configurar algunas opciones extra. Se sugiere seleccionar la opción "Enable file system caching", que mejora el rendimiento en repos grandes. Luego de esta configuración, hacer clic para instalar Git.

16. Verifica la instalación. Abre Git Bash o Windows Power Shell y ejecuta:

git --version

Cuando se trabaja en equipos compartidos (como las computadoras de Key institute), es importante que cada estudiante pueda enlazar Git con su cuenta personal de GitHub de manera segura, sin dejar credenciales almacenadas permanentemente en la máquina. Estos pasos, seguirán esta perspectiva. Sin embargo, los usuarios que tienen su computadora personal, pueden seguir estos pasos perfectamente.

1. Lo primero es crear un "Token", que permitirá conectar la cuenta de Github en el entorno de Git. Se utiliza este método porque es mucho más seguro que el uso de contraseñas. Para crear el Token, en github, haz clic en tu avatar (arriba a la derecha) y selecciona "Settings" (Configuración).

2. En el menú lateral izquierdo, baja hasta la sección "Developer settings" (al final). Dentro de "Developer settings", haz clic en "Personal access tokens".

3. Elige la opcion "Fine-grained tokens", que son la versión más actual y segura, permite configurar permisos más detallados por repositorio. Luego haz clic en "Generate new token".

4. Llena el formulario del token.

  • Token name: nombre del token, en este caso se ha utilizado "key-fundamentos-programacion-temp"
  • Descripción: campo que puede ser utilizado para contextualizar el token. No es obligatorio llenarlo.
  • Resourse owner: usuario
  • Expiration: tiempo de vida del token, para computadoras personales, es posible definir "no expiration", para computadoras públicas, se sugiere configurar alguna de las opciones entre 7 y 30 días.
  • Repository Access: Seleccionar la opción "Only select repositories" y seleccionar el repositorio de fundamentos de programación
  • En la sección de permisos, haz clic en "Repository permissions" y ubica la opción "Contents", que debes configurar como "Read and write". Este permiso habilitará la opción de actualizar el repositorio desde nuestros recursos locales.
  • Luego de aplicar la configuración, haz clic en "generate token".

5. El siguiente paso es confirmar en una ventana emergente la configuración aplicada.

6. A continuación, se generará una cadena de texto. ¡Importante! debemos copiar y almacenar esta cadena de texto porque será necesaria para poder conectar git con github y en este paso es el único momento que tenemos acceso a esta cadena de texto y podemos copiarla.

Clonar un repositorio tiene varias utilidades importantes, especialmente cuando quieres agregar archivos o contribuir a un proyecto. Cuando clonas un repositorio, obtienes una copia completa del proyecto en tu computadora. Esto te permite:

  • Trabajar sin conexión.
  • Explorar el código cómodamente.
  • Editar, agregar o eliminar archivos.

Clonar el repositorio permite modificar el código, crear nuevos archivos o agregar funcionalidades sin afectar el repositorio original (hasta que decidas subir tus cambios). Si estás trabajando con otros programadores, puedes hacer cambios localmente de forma segura, subir tus aportes y actualizar el proyecto cuando estes seguro que el cambio esta correcto. Para clonar tu repositorio sigue los siguientes pasos:

1. Ir al repositorio y en el botón "Code" copiar la URL del repositorio

2. En nuestro sistema operativo, ubicar un directorio para almacenar la carpeta del repositorio. Se sugiere crear una carpeta en la raíz (en el caso de Windows en el disco C). En este ejemplo se ha creado manualmente (clic derecho > nuevo > nueva carpeta) llamada "Proyectos". Entrar al directorio y hacer clic derecho, en el menú emergente, ubicar la opción "abrir una terminal".

¡IMPORTANTE!

Cuando trabajes en una computadora pública (cómo el aula CLIC), compartida (cómo el servicio de préstamo de Key) o prestada (es un equipo de algún familiar o amigo), es fundamental configurar tu nombre de usuario y correo electrónico en Git usando los comandos git "config user.name" y "git config user.email". Esto asegura que tus contribuciones queden correctamente registradas con tu identidad y evita confusiones con el trabajo de otros estudiantes. Además, permite que tus commits se asocien a tu cuenta de GitHub y aparezcan como parte de tu actividad. Si omites este paso, podrías perder el crédito de tu trabajo o generar conflictos en el historial del repositorio. En palabras simples: cada vez que vayamos a actualizar un repositorio desde una computadora pública, compartida o prestada, será necesario configurar estos metadatos.

Recomendación: El proceso de limpieza de credenciales es opcional en equipos personales, pero debe considerarse obligatoria en entornos compartidos para proteger tu identidad y tu repositorio.

Si estás utilizando una computadora pública o compartida temporalmente, es fundamental eliminar las credenciales guardadas (como tu token personal de GitHub) al finalizar tu sesión de trabajo. Esto evita que otra persona pueda subir cambios a tu repositorio usando tu identidad sin autorización.

Si utilizas el servicio de préstamo de equipos de Key Institute, el proceso de limpieza de credenciales lo puedes realizar cuando termine el periodo de préstamos y tengas que devolverlo.

Para eliminar el token guardado en Git, puedes ejecutar el siguiente comando en la terminal:

git credential-manager erase

A continuación el promt queda en espera de los parámetros de eliminación de credenciales, por lo que debes escribir los siguiente:

protocol=https
host=github.com

Y presiona Enter dos veces.

Esto eliminará de forma segura el token almacenado para GitHub, y Git volverá a pedir autenticación la próxima vez que intentes hacer git push.
